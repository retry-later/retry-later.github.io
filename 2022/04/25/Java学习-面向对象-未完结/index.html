关于对象属性的一些细节

  • 在叫法上：成员变量 = 属性 = field（字段）
  • 属性是类的重要组成部分，可以是基本的数据类型（int、double）也可以是引用类型（对象，数组、字符串）
  • 属性的定义语法规则：和变量的定义规则是一样的，但是多了一个访问修饰符；
  • 如果属性只定义了并没有赋值，属性是有默认值的。默认值的规则和数组的默认值规则一致。int 0,short 0,byte 0; float 0.0;double 0.0;char \u0000;boolean false;String null;

类的两种创建形式

  • 先声明在创建
  • 直接创建

Tips：因为对象也是引用类型，所以Person p1 = new Person(); p1.name = "小明"; p1.age = 10; p2 = p1; p2.age = 100; 和数组一样，对p2的修改也会影响p1。因为他们两个指向的是同一个对象。当同一个类被多次实例化的时候，类信息在方法区只会加载一次，不会重复的加载。

成员方法

方法调用机制

  • 当程序执行到方法的时候会在栈空间中开辟一个独立的栈空间，
  • 当方法执行结束之后，或者执行到return语句就会返回，将返回到调用方法的地方。同时第一步开辟出的栈空间将会被回收。
  • 返回后执行方法后面的代码
  • 当main方法（程序的入口方法）执行完毕之后，栈空间的main方法所属的栈也会被回收。同时整个程序退出。

方法的好处

  • 提高代码的复用
  • 将实现的细节封装起来，共其他程序进行调用。

成员方法的定义

方法的注意事项和细节

  • 访问修饰符，控制方法的可用范围；共有四种修饰符（default、private、public、protected），不写即为默认的访问修饰符。
  • 一个方法最多只能够有一个返回值，如果要返回多个值可以返回一个数组。
  • 返回类型可以为任意类型，包含基本类型或者引用类型
  • 如果方法要求有返回类型，则方法体中必须包含return语句。并且返回的数据类型要和要求的数据类型一致或者兼容。
  • 如果方法的返回类型为void，可以不写return或者只写return（后面不能跟任何数据）。
  • 一个方法可以有0个或者多个参数，多个参数中间使用都好隔开。
  • 参数的类型可以是任何类型，可以是基本数据类型也可以是引用类型。
  • 调用带参数的方法是，传入的参数一定对应参数列表传入同类型参数或者相兼容的参数。
  • 方法体中不能够再定义方法。

方法的调用

  • 同一个类中的方法调用，不必实例化对象。直接函数名调用即可。
  • 跨类中的方法，A类中调用B类的方法需要通过对象名进行调用。比如在上面的示例中MethodDetail中调用A的sea yOK需要先new一个A的对象，之后通过对象名进行调用。
  • 跨类的方法调用和方法的访问能修饰符有重要的关联。

练习题：

方法的传参

  • 对于基本数据类型，传递的是值（值拷贝），形参的任何改变不影响实参。
  • 对于引用类型，形参的改变也会影响实参。（比如数组、对象）

方法递归调用

介绍：简单的说递归就是方法自己调用自己，每次调用时传入不同的变量，递归有助于编程者解决复杂问题，同时可以让代码变得更加简洁。

递归方法调用的规则：

  • 执行一次方法就会创建一个新的受保护的独立空间（栈空间）
  • 方法的局部变量是独立的，不会相互影响。
  • 如果方法中使用的是引用类型变量（比如数组，对象），就会共享该引用类型的数据。
  • 递归必须向退出递归的条件逼近，否则就会出现无限的递归。
  • 当一个方法执行完毕或者遇到return，就会返回遵守谁调用，就将结果返回给谁，体哦那个事当方法执行完毕或者返回时，该方法也就执行完毕了。

方法的可变参数

  • 概念：Java允许将同一个类中==多个同名同功能==但参数个数不同的方法封装成一个方法。

  • 语法：

  • 使用：使用可变参数时可以作为数组进行操作。

  • 细节：

    • 可变参数的实参可以是0个或者任意个数
    • 可变参数的实惨可以是数组

    • 可变参数的本质就是数组

    • 可变参数可以和普通类型的参数一起放到参数列表中，但必须保证可变参数在最后

变量的作用域

基本使用：

  • Java编程中主要的变量就是属性（成员变量）和局部变量
  • 局部变量一般是指成员方法中定义的变量
  • Java中作用域的分类：全局变量（也就是属性）作用域为整个类；局部变量：也就是除了属性之外的其他变量。作用域为定义它的代码块。
  • ==全局变量可以不赋值直接使用，因为它有默认值。但是局部变量必须赋值后才能够使用，因为没有默认值==。

细节：

  • 属性和局部变量可以重名，访问时遵循就近原则
  • 在同一个作用域中，比如在同一个成员方法中，两个局部变量不能重名。
  • 属性生命周期比较长，伴随着对象的创建而创建。伴随着对象的消亡而消亡。局部变量，生命周期短，伴随着代码块的执行而创建，伴随代码块的消亡而消亡。
  • 作用域的范围不同
  • 修饰符不同，属性可以设置修饰符、但是局部变量不能设置修饰符。

构造器（构造方法）

基本语法：

  • 构造器没有返回值，构造方法的修饰符可以是默认的，也可以是public、protected、private
  • 方法名必须和类名一致
  • 参数列表和成员方法一样的规则
  • 构造器的调用系统完成。
  • Java类有一个默认的无参构造器，如果自己定义了那么默认的无参构造器就不可以用了。也就是Dog tom = new Dog();这种实例化方式不能够在用了，如果仍想使用无参构造器的话可以在定义一个无参构造器，形成构造器的重载。

this关键字的使用细节

  • this用于访问本类的属性、方法、构造器
  • this用于区分当前类的属性和局部变量
  • 访问成员方法的语法为this.function(args)；
  • 访问构造器语法this(参数列表)，注意只能在构造器中使用（也就是只能在构造器中访问另一个构造器，必须放到第一条语句中）
  • this不能在类定义的外部使用，只能在类定义的方法中使用。

Java包

Java包的三大作用：

  • 区分相同名字的类
  • 当类很多时，可以很好的管理类
  • 控制访问范围

包的基本语法

package com.test;

包的命名规范

  • 命名规则
    • 只能包含数字、字母、下划线、小圆点，不能够以数字开头，不能是关键字或者保留字。
  • 命名规范
    • 一般是小写字母+小圆点+业务模块名称（com.公司名.项目名.业务模块名）
    • com.sina.crm.utils工具类

访问控制符

Java提供了四种级别的访问控制符号，用于控制方法和属性（成员变量）的访问范围，

  1. 公开级别：public修饰，对外公开。
  2. 受保护的级别：protected修饰，对子类和同一个包中的类公开。
  3. 默认级别：没有修饰符号，向同包的类公开
  4. 私有级别：private 修饰，只有类本身可以访问，不对外公开。

使用注意细节：

  1. 修饰符可以用来修饰类的属性，成员方法以及类。
  2. 只有默认和public才能对类进行修饰，并且遵循上述访问权限的特点
  3. 成员方法的访问规则和属性完全一样。

封装、继承、多态

封装

encapsulation，就是把抽象的数据[属性]和对数据的操作[方法]封装在一起，数据被保护在内部，程序的其他部分只能通过被授权的操作才能对数据进行操作。

继承的细节

  • 子类继承了父类的所有属性和方法，但是私有的属性和方法不能够在子类中直接访问，要通过公开方法间接访问。
  • 子类必须调用父类的构造器完成父类的初始化
  • 当创建子类对象时，不管使用子类的那个构造器，默认情况下总会去调用父类的无参构造器，如果父类没有提供无参构造器则必须在子类的构造器中使用super指定父类的初始化工作，否则编译将不会通过。
  • 如果希望指定去调用父类的某个构造器，则显示的调用一下：super(参数列表)
  • super在使用时，需要放到构造器的第一行 super只能在构造器中使用。
  • super()和this()都只能放在构造器的第一行，因此两个方法不能共存在一个构造器中

多态

对象的多态：

  • 一个对象的编译类型和运行类型可以不一致
  • 编译类型在定义对象时，就确定了，不能改变。
  • 运行类型是可以改变的。
  • 编译类型看定义时=号左边的，运行类型看=号右边的。

因为类时引用类型，所以在运行时的变量类型需要看变量指向堆内存的对象地址。所以变量指向那种类型他就是哪种类型。一个父类的对象类型可以指向一个子类的对象的。

向上转型

  • 本质：父类的引用指向了子类的对象
  • 语法：父类类型 引用名 = new 子类类型()
  • 特点：编译类型看左边，运行类型看右边。可以调用父类中的所有成员（须遵守访问权限），不能调用子类中的特有成员。
  • 最终效果看子类的具体实现。

向下转型

  • 语法 子类类型 引用名 = （子类类型）父类引用
  • 只能转换父类的引用，不能强转父类的对象
  • 要求父类的引用必须指向当前目标类型的对象这里这个前提是指这个父类的引用，之前指向的就是一个子类的对象
  • 当向下转型后，可以调用子类类型中的所有成员

属性

属性的值只看编译类型，与运行类型无关。

instanceof

用来判断对象的运行类型是否为XX类型或者XX的子类

Java的动态绑定机制（重点）

  • 当调用对象方法的时候，该方法会和该对象的内存地址/运行类型绑定。
  • 当调用对象属性的时候，没有动态绑定机制。哪里声明，哪里使用。

多态的应用

多态数组

  • 数组的定义类型为父类类型，里面保存的实际元素为子类类型

Object

Object是类层次结构的根类，每个类都适用Object作为超类（包括数组）都实现这个类的方法。
